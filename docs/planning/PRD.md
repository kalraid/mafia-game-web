## 1. 프로젝트 개요

### 1.1 제품 비전
혼자서도 완전한 마피아 게임 경험을 할 수 있도록, 나머지 플레이어 자리를 **독립적인 AI Agent**로 채우는 웹 기반 마피아 게임.  
각 AI Agent는 자신의 진영(시민/마피아/중립)의 이익을 위해 **자율적으로 판단하고 행동**하며, 실제 사람과 구분되지 않을 수준의 자연스러운 대화를 지향한다.

### 1.2 핵심 목표
| # | 목표 | 설명 |
|---|------|------|
| 1 | **몰입감 있는 AI 대화** | AI 플레이어들이 서로, 그리고 사람과 자연스럽게 채팅 |
| 2 | **룰 자동 진행** | 낮/밤 사이클, 투표, 능력 사용 등 게임 진행 자동화 |
| 3 | **전략적 AI** | 각 진영 슈퍼바이저가 팀 전략을 수립하고 각 Agent에 지시 |
| 4 | **실시간 웹 UI** | WebSocket 기반 채팅 + 상태창 동시 표시 |

### 1.3 사용자 정의
- **주 사용자**: 마피아 게임을 혼자 즐기고 싶은 1인 플레이어
- **게임 참여**: 사용자 1명 + AI Agent N-1명 (최대 20인)

## 2. 기술 스택 개요

```
Frontend   : Streamlit (WebSocket 기반 실시간 UI)
Backend    : FastAPI (단일 서버)
통신       : WebSocket (FastAPI 기반)
AI 프레임워크: LangChain + LangGraph (ReAct 패턴)
지식 검색   : RAG (ChromaDB + sentence-transformers)
에이전트 통신: MCP (Model Context Protocol) + A2A (Agent-to-Agent)
LLM        : Claude API (claude-sonnet-4 계열)
세션 메모리 : Redis + LangGraph Checkpointer (필수)
```

## 3. 구현 범위 정의

### MVP 범위 (반드시 구현)
- 게임 엔진 코어 (Phase, 타이머, 투표, 직업 능력)
- AI Agent 기초 (ReAct 패턴, 페르소나, 발언/투표)
- WebSocket 실시간 채팅
- Streamlit 기본 UI (채팅창 + 상태창)
- 슈퍼바이저 + A2A 지시 체계
- Redis Checkpointer (멀티턴 메모리)

### 2단계 범위 (시간 허용 시)
- RAG 지식베이스 구축 및 연동
- MCP Tool 서버 완성
- UI 스타일링 (낮/밤 테마, 사망 오버레이)
- 발언 타이밍 자연화 (랜덤 딜레이)

### 로드맵 (추후)
- 게임 결과 저장 및 리뷰 기능
- 멀티 게임 세션 지원
- 모바일 반응형 UI

## 4. 핵심 기능 목록 (Feature List)

### 4.1 게임 설정 & 로비
- [🔄] 플레이어 수 선택 (4~20명)
- [🔄] 사용자 닉네임 입력
- [🔄] 직업 구성 설정 (자동 or 수동)
- [🔄] 게임 시작 버튼

### 4.2 AI Agent 시스템
- [🔄] 각 AI 플레이어별 독립 Agent 인스턴스 (LangGraph ReAct)
- [🔄] 시민 슈퍼바이저 Agent
- [🔄] 마피아 슈퍼바이저 Agent
- [🔄] 중립 슈퍼바이저 Agent
- [🔄] A2A 통신: 슈퍼바이저 ↔ 하위 Agent 지시 채널
- [⬜] MCP Tool: 게임 상태 조회, 능력 사용, 투표 등록
- [⬜] RAG: 마피아 전략 지식베이스 활용
- [⬜] Redis Checkpointer: Agent 멀티턴 메모리

### 4.3 게임 진행 (Game Loop)
- [🔄] 낮 Phase: 전체 채팅 (3분) → 투표 → 처형
- [🔄] 밤 Phase: 직업별 능력 사용 → 결과 집계
- [🔄] 승리 조건 판정 자동화
- [⬜] 게임 로그 저장

### 4.4 채팅 시스템
- [🔄] 실시간 WebSocket 채팅
- [🔄] 낮/밤에 따른 채팅 채널 분리 (전체 / 마피아 전용)
- [⬜] AI 발언 딜레이 (자연스러운 타이밍)
- [🔄] 시스템 메시지 (게임 이벤트 알림)

### 4.5 UI
- [🔄] 좌측 3/4: 채팅 영역
- [🔄] 우측 1/4: 상태창 (밤낮 표시 / 플레이어 목록 / 버튼)
- [⬜] 사망 플레이어: 검은 천 오버레이

## 8. 사용자 요구사항 원문 정리 (User Input)

> ⚠️ **이 섹션은 사용자가 직접 언급한 내용을 원문 보존한 것입니다.**

```
- streamlight + fastAPI 로 구성
- socket 기반의 통신을 통해서 채팅형식
- 플레이어수 지정 (최대 20), 접속한 사람 제외 나머지는 AI (각각 하나의 agent)
- 메인 슈퍼바이저: 시민 슈퍼바이저, 마피아 슈퍼바이저, 중립 슈퍼바이저
  → 각각 자기들 팀이 이기기 위해 해야할 일들을 매번 고민
- 낮(3분 대화 → 투표 → 처형) / 밤(직업별 능력 사용 → 사망 처리)
- 마피아들은 각각 한명씩 쏠수 있고 낮이 되면 지정된 플레이어 사망
- 마피아수 >= 시민수 → 마피아 승리 / 마피아 전원 제거 → 시민 승리
- 화면: 살아있는 플레이어 표시, 죽으면 검은 천 오버레이
- 화면 구성: 좌 3/4 채팅창, 우 1/4 상태창
- 상태창: 위 1/5 밤낮표시 / 중간 3/5 플레이어 표시 / 하단 1/5 버튼
- 기술: langchain, langgraph, rag, mcp, a2a 적극 활용
- 백엔드 서버 3~4개 가능, 가능하면 1개 권장
```
