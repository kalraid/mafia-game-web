## 1. 화면 구성 개요

전체 화면은 **좌 3/4 채팅 영역** + **우 1/4 상태창**으로 구성된다.

```
┌──────────────────────────────────────┬──────────────┐
│                                      │              │
│                                      │  [밤/낮 표시] │  ← 1/5
│                                      │  ◐ 낮 2라운드 │
│                                      │  ⏱ 02:14     │
│          채팅 영역                    ├──────────────┤
│          (3/4)                       │              │
│                                      │  [플레이어   │
│                                      │   목록]      │  ← 3/5
│                                      │              │
│                                      │  👤 김민준 ✅ │
│                                      │  👤 박서연 ✅ │
│                                      │  👤 이지호 💀 │
│                                      │  👤 최수아 ✅ │
│                                      │  ...         │
│                                      ├──────────────┤
│                                      │  [버튼 영역]  │  ← 1/5
│                                      │  [투표] [능력]│
│                                      │  [기권] [설정]│
└──────────────────────────────────────┴──────────────┘
```

## 2. 채팅 영역 상세 (좌 3/4)

### 2.1 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  AI Mafia Online                    🌞 낮 2라운드   │  ← 헤더
├─────────────────────────────────────────────────────┤
│                                                      │
│  [시스템] 밤이 밝아왔습니다. 이지호 님이 사망했습니다. │
│  이지호의 직업: 시민                                 │
│                                                      │
│  👤 김민준  02:12  ─────────────────────────────    │
│  │ 어젯밤에 이지호가 죽었네... 난 이지호 믿었는데    │
│  │ AI_4(박서연) 수상한 거 아님?                     │
│                                                      │
│  👤 박서연  02:15  ─────────────────────────────    │
│  │ 저 왜 의심하세요? 저 그냥 시민이에요...           │
│                                                      │
│  🟢 나 (플레이어)  02:18  ────────────────────────  │
│  │ 저는 이번에 최수아를 의심합니다.                  │
│                                                      │
│  ···                                                 │
│                                                      │
├─────────────────────────────────────────────────────┤
│  💬 메시지를 입력하세요...              [전송] ⌨️   │  ← 입력창
└─────────────────────────────────────────────────────┘
```

### 2.2 메시지 타입별 스타일

| 타입 | 배경색 | 아이콘 | 설명 |
|------|--------|--------|------|
| 시스템 메시지 | 노란색 배경 | ⚠️ | 게임 이벤트 알림 |
| AI 플레이어 발언 | 회색 | 👤 | AI 채팅 |
| 플레이어(나) 발언 | 파란색 | 🟢 | 사용자 채팅 |
| 마피아 비밀 채팅 | 빨간색 | 🔴 | 마피아만 보임 |
| 사망자 메시지 | 어두운 회색 + 취소선 | 💀 | 사망 후 표시 |

### 2.3 채팅 동작 규칙
- 낮 Phase: 입력창 활성화, 전송 가능
- 밤 Phase (시민): 입력창 비활성화, 읽기 전용
- 밤 Phase (마피아): 마피아 비밀 채팅 채널 전환, 입력 가능
- 투표 Phase: 입력창 비활성화
- 메시지 자동 스크롤: 새 메시지 도착 시 최하단 이동

## 3. 상태창 상세 (우 1/4)

### 3.1 밤/낮 표시 영역 (상단 1/5)

```
┌──────────────────┐
│  ☀️  낮  2라운드  │
│   ──────────     │
│   ⏱  02:14      │
│   [=====----]    │  ← 타이머 프로그레스바
└──────────────────┘
```

| 상태 | 아이콘 | 배경색 |
|------|--------|--------|
| 낮 채팅 | ☀️ | 밝은 노란색 |
| 낮 투표 | 🗳️ | 주황색 |
| 최후 변론 | 💬 | 오렌지 |
| 밤 | 🌙 | 짙은 파란색 |
| 결과 발표 | 🔔 | 보라색 |

### 3.2 플레이어 목록 영역 (중간 3/5)

```
┌──────────────────┐
│  플레이어 (8/10) │
│                  │
│  👤 김민준       │  ← 생존 플레이어
│  👤 박서연       │
│  💀 이지호       │  ← 사망: 검은 천 오버레이
│     [   사망   ] │
│  👤 최수아       │
│  👤 정태양       │
│  ...             │
│                  │
│  🟢 나 (플레이어)│  ← 사용자 구분 표시
└──────────────────┘
```

**플레이어 카드 상태**:
- 생존: 이름 + 아바타 표시
- 사망: 검은 반투명 오버레이 + "사망" 텍스트 + 직업 공개
- 내 카드: 초록색 테두리 강조
- 의심 표시: 투표 많이 받을수록 카드에 빨간 테두리 (투표 수 비례)

**플레이어 카드 클릭 시**:
- 투표 Phase: 해당 플레이어에게 투표
- 능력 사용 Phase: 해당 플레이어를 능력 대상으로 선택
- 일반 Phase: 해당 플레이어 채팅 히스토리 하이라이트

### 3.3 버튼 영역 (하단 1/5)

**Phase별 표시 버튼**:

| Phase | 버튼 |
|-------|------|
| 낮 채팅 | `[기권하기]` `[게임 설정]` |
| 낮 투표 | `[투표 완료]` `[기권(무투표)]` |
| 최후 변론 | `[처형 찬성]` `[처형 반대]` |
| 밤 (마피아) | `[공격 대상 확정]` |
| 밤 (경찰) | `[조사 확정]` |
| 밤 (의사) | `[보호 확정]` |
| 밤 (시민) | `[대기 중...]` (비활성) |

## 6. Streamlit 구현 고려사항

### 6.1 실시간 업데이트
- `st.empty()` + loop로 채팅/상태 실시간 갱신
- WebSocket 메시지 수신 시 `st.rerun()` 트리거
- 또는 `streamlit-autorefresh` 컴포넌트 활용 (500ms 간격)

### 6.2 커스텀 CSS
- 검은 천 오버레이: CSS `::after` pseudo-element
- 채팅 버블 스타일: `st.markdown()` + HTML 인라인
- 타이머 프로그레스바: `st.progress()` 활용

### 6.3 JavaScript 연동
- WebSocket 클라이언트: `streamlit-javascript` 또는 커스텀 컴포넌트
- 타이머 실시간 카운트다운: JS 인터벌 사용
- 알림음: JS Web Audio API

### 6.4 모바일 대응
- 초기 버전: 데스크탑 전용 (최소 1280px 너비)
- 추후 반응형 레이아웃 적용 고려
